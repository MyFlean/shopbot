Flean API Documentation

Complete REST API documentation for the Flean mobile application. Search healthy products, browse categories, and scan product labels.

Get Started View Models
Base URL https://api.flean.ai/rs

Overview

The Flean API provides endpoints for product discovery, search, and health analysis for the Flutter mobile app. All responses are JSON formatted.

API Flow

HOME SCREEN ├── Banners → GET /api/v1/home/banners ├── Categories → GET /api/v1/home/categories ├── Best Selling → GET /api/v1/home/best-selling → [Product Cards] ├── Curated → GET /api/v1/home/curated → [Product Cards] ├── Why Flean → GET /api/v1/home/why-flean └── Collabs → GET /api/v1/home/collaborations SEARCH / CATALOGUE (Unified - Recommended) └── POST /api/v1/products → [Product Cards] (supports query, subcategory, filters, pagination, sorting) [Product Card] tapped ├── GET /api/v1/product/{id} → Full PDP └── GET /api/v1/product/{id}/recommended → [Product Cards] SCANNER ├── POST /api/v1/scanner → Top 3 [Product Cards] ├── User selects one → GET /api/v1/product/{id} → Full PDP └── GET /api/v1/product/{id}/alternatives → 5 [Product Cards]

Authentication

No Authentication Required

The Flean API currently does not require authentication. All endpoints are publicly accessible.

Response Format

All API responses follow a consistent JSON structure.

Success Response

JSON 
{
  "success": true,
  "data": {
    // Response data here
  },
  "meta": {
    // Pagination, timestamps, etc.
  }
}

Error Response

JSON 
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable description"
  }
}

GET /api/v1/home/categories

Returns category grid for the home screen. By default returns 4 categories; use ?all=true for all categories.

Query Parameters

Parameter Type Required Description

Unified Products API

The recommended endpoint for both Search and Catalogue screens with full pagination and filter support.

POST /api/v1/products
Recommended

Unified product search with text query, subcategory filtering, pagination, and filters. Use this for both Search Screen and Catalogue Screen for consistent behavior.

Why use this endpoint?

Combines the capabilities of /search (filters) and /catalogue (pagination) into a single, consistent API.

Request Body

Parameter Type Required Description
query string Optional* Text search query (e.g., "protein bars")
subcategory string Optional* ES category path (e.g., "f_and_b/food/light_bites/chips")
page integer Optional Page number, 0-indexed (default: 0)
size integer Optional Items per page, 1-100 (default: 20)
sort_by string Optional Sort option (see below)
filters object Optional Filter options (see below)

* At least one of query or subcategory must be provided.

Sort Options

Value Description
relevanceDefault - ES relevance score
price_ascPrice: Low to High
price_descPrice: High to Low
protein_descProtein: High to Low
fiber_descFiber: High to Low
fat_ascFat: Low to High

Filter Options

Filter Type Values
price_range string below_99, 100_249, 250_499, above_500
flean_score string 10, 9_plus, 8_plus, 7_plus
preferences array no_palm_oil, no_added_sugar, no_additives
dietary array dairy_free, gluten_free
cURL 
curl -X POST "https://api.flean.ai/rs/api/v1/products" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "protein bars"
  }'
cURL 
curl -X POST "https://api.flean.ai/rs/api/v1/products" \
  -H "Content-Type: application/json" \
  -d '{
    "subcategory": "f_and_b/food/light_bites/chips_and_crisps",
    "page": 0,
    "size": 20
  }'
cURL 
curl -X POST "https://api.flean.ai/rs/api/v1/products" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "chips",
    "sort_by": "price_asc",
    "filters": {
      "price_range": "below_99",
      "flean_score": "8_plus",
      "preferences": ["no_palm_oil"],
      "dietary": ["gluten_free"]
    }
  }'
JSON 
{
  "success": true,
  "data": {
    "products": [
      // Array of Product Cards
    ]
  },
  "meta": {
    "total": 245,
    "page": 0,
    "size": 20,
    "total_pages": 13,
    "has_next": true,
    "has_prev": false,
    "query": "chips",
    "subcategory": null,
    "sort_by": "price_asc",
    "filters_applied": {
      "price_range": "below_99",
      "flean_score": "8_plus"
    }
  }
}
POST /search
Legacy

Legacy search endpoint. Supports filters but no pagination.

Deprecated

This endpoint does not support pagination. Consider migrating to POST /api/v1/products.

Example Request

cURL 
curl -X POST "https://api.flean.ai/rs/search" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "protein bars",
    "sort_by": "protein_desc",
    "filters": {
      "price_range": "100_249"
    }
  }'

GET /api/v1/product/{product_id}/alternatives

Returns up to 5 healthier alternatives in the same subcategory, sorted by Flean percentile (healthiest first).

Example Request

cURL 
curl -X GET "https://api.flean.ai/rs/api/v1/product/01K1B1BPGN2WAXFB5DNSGXX4W3/alternatives"
GET /api/v1/product/{product_id}/recommended

Returns recommended products for the PDP "You May Also Like" section.

Query Parameters

Parameter Type Required Description
limit integer Optional Number of products (1-10, default: 8)

Example Request

cURL 
curl -X GET "https://api.flean.ai/rs/api/v1/product/01K1B1BPGN2WAXFB5DNSGXX4W3/recommended?limit=6"

Scanner API

Scan product images to identify and find healthier alternatives.

POST /api/v1/scanner

Scan a product image to identify it and return top 3 matching product cards. Uses Claude Vision for OCR.

Scanner Flow

Step 1: POST /api/v1/scanner → Top 3 product cards Step 2: User selects a product Step 3: GET /api/v1/product/{id} → Full PDP Step 4: GET /api/v1/product/{id}/alternatives → 5 healthier options

Request Body

Parameter Type Required Description
image string Required Base64-encoded image or data URL (max 5MB)

Supported Formats

  • JPEG / JPG
  • PNG
  • GIF
  • WebP

Example Request

cURL 
curl -X POST "https://api.flean.ai/rs/api/v1/scanner" \
  -H "Content-Type: application/json" \
  -d '{
    "image": "data:image/jpeg;base64,/9j/4AAQSkZJRg..."
  }'

Error Handling

Standard error codes and their meanings.

Code HTTP Description
MISSING_IMAGE 400 Scanner: No image provided
INVALID_IMAGE 400 Scanner: Invalid format or exceeds 5MB
MISSING_PARAMETER 400 Products: Neither query nor subcategory provided
INVALID_FILTERS 400 Invalid filter values
PRODUCT_NOT_FOUND 404 Product ID not found
INTERNAL_ERROR 500 Internal server error
